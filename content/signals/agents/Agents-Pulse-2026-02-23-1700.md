### Agentic Infrastructure | MCP Security Vulnerabilities Disclosed — Prompt Injection via Sampling Feature

- **What**: 
Palo Alto Unit42 examined MCP's sampling feature in a widely used coding copilot and showed that without proper safeguards, malicious MCP servers can exploit sampling for a range of attacks, demonstrating risks through three proof-of-concept examples.
 
MCP sampling introduces prompt injection as a primary attack vector; the protocol's design allows MCP servers to craft prompts and request completions from the client's LLM, enabling injection of hidden instructions that can manipulate outputs and influence subsequent tool executions.

- **Why it matters**: 
Security researchers demonstrated how agents with broad permissions can be manipulated through malicious prompts or web content, with some instances publicly accessible.
 As MCP becomes universal infrastructure, its attack surface scales proportionally. This is the "Log4Shell moment" risk for agentic infrastructure.

 Three simultaneous inflection points are now converging:

**1. Protocol consolidation is complete (or nearly so).** 
LangGraph reached 1.0 in October 2025, CrewAI passed 450 million processed workflows, Amazon Bedrock AgentCore launched as a managed deployment platform, and the Model Context Protocol became the de facto standard for tool integration.
 The framework wars are not over, but the tool-integration standard war is. MCP won.

**2. The production gap is narrowing — but governance debt is accumulating fast.** 
The shift from "assistant" to "operator" is not semantic — it is architectural.
 Enterprises that deployed quickly are now discovering that 
the critical variable is not raw intelligence but whether institutions build the surrounding systems — technical and social — that make autonomous AI stable and accountable.


**3. Framework stratification is hardening.** 
Three frameworks now dominate production agent orchestration with distinct philosophies: LangGraph (graph-state, durable execution, human-in-the-loop), CrewAI (role-based teams), and OpenAI Agents SDK (lightweight, semi-managed).
 Practitioners are moving away from "which framework exists" to "which layer of the stack does my problem live in." The stack has matured enough to stratify.

One significant concern: 
Forrester predicts that 75% of firms attempting to build complex agentic architectures on their own will fail — the systems are too convoluted.
 This is a major tailwind for managed platforms and SI integrators.
